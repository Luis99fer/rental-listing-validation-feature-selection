# Project Overview

## Project Name

Rental Listing Validation, Feature Selection and Hyperparameter Optimization

## Overview

This project is a machine learning case study based on the Kaggle competition **Two Sigma Connect: Rental Listing Inquiries**.

The objective is to predict the level of interest generated by rental apartment listings using structured listing attributes, text-derived amenity features, location data, price information, and listing metadata.

The repository focuses on validation strategy, cross-validation, feature selection, hyperparameter optimization, regularization, and model comparison.

## Business Problem

Rental platforms receive thousands of apartment listings. Some listings attract high user interest, while others receive low engagement.

The goal of this project is to build a machine learning workflow that can estimate listing interest level and help understand which features contribute most to user demand.

This type of model can support:

```text
Rental listing ranking
Demand prediction
Listing quality analysis
Pricing strategy
Feature importance analysis
Model validation strategy
```

## Machine Learning Task

The task is a supervised prediction problem.

The target variable is:

```text
interest_level
```

Typical classes are:

```text
low
medium
high
```

The project focuses on preprocessing, validation design, feature selection, and hyperparameter tuning rather than only training one final model.

## Validation Scope

The project implements and compares several validation schemes.

Custom splitting methods include:

```text
Random train-test split
Random train-validation-test split
Date-based train-test split
Date-based train-validation-test split
Deterministic splitting
```

Custom cross-validation methods include:

```text
K-Fold
Grouped K-Fold
Stratified K-Fold
Time Series Split
```

The notebook also compares custom implementations with scikit-learn validation utilities.

## Feature Selection Scope

The project explores several feature selection approaches.

Implemented and evaluated methods include:

```text
Lasso coefficient ranking
Missing-value ratio filtering
Correlation-based filtering
Permutation importance
SHAP-based interpretation
```

The selected features are used to refit models and compare quality, speed, and stability.

## Hyperparameter Optimization Scope

The project evaluates different hyperparameter optimization strategies for ElasticNet.

Implemented methods include:

```text
Grid Search
Random Search
Bayesian optimization with Optuna
Optuna with cross-validation
```

The goal is to compare search strategies and understand how hyperparameter tuning affects model performance.

## Main Workflow

The notebook follows this workflow:

```text
Load rental listing data
Perform preprocessing and feature engineering
Create amenity-based binary features
Implement custom splitting methods
Implement custom cross-validation methods
Compare custom validation schemes with scikit-learn
Apply feature selection methods
Train models using selected feature subsets
Tune ElasticNet hyperparameters
Compare grid search, random search, and Optuna
Analyze model quality and overfitting behavior
Summarize final results
```
